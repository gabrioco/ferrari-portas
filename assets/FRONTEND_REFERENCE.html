03 — Backgrounds & Surfaces
Noise Overlay
SVG fractal noise at 4–8% opacity. Adds texture to flat surfaces. Apply as fixed overlay, z-index high, pointer-events:none.
Grid Background
CSS linear-gradient grid at 40px. Use .grid-bg (dark) or .grid-bg-light (light theme). Subtle structural depth.
Glass Panel (Dark)
rgba(23,23,23,0.6) + backdrop-filter:blur(20px). Essential for floating navs, modals, card overlays.
Glass Card (Dark Subtle)
rgba(255,255,255,0.04) + blur(12px). Lighter glass for card surfaces. Pair with border rgba(255,255,255,0.06).
Border Gradient (CSS Mask)
Gradient border via CSS mask-composite. Add data-border-gradient attr and set --border-gradient CSS var. No extra element needed — uses ::before.
Gradient Blur (Layered Backdrop)
6-layer backdrop-filter blur with gradient masks. Use for nav top bars — content scrolls through without hard line.

08 — Usage Rules for Claude Code
Non-negotiable
  • → Always use CSS custom properties (--bg, --text, --accent) for theming. Never hardcode hex directly in components.
  • → Never default to Arial, Roboto, or system fonts. Always pick from the pairs above.
  • → Never use purple gradients on white — it is the most overused AI-generated aesthetic.
  • → All entrance animations must use either fadeSlideIn or textSlide keyframes. No generic opacity fades alone.
  • → Card borders: rgba(255,255,255,0.08) on dark, rgba(0,0,0,0.08) on light. Never solid 1px #ccc.
  • → Typography labels/tags: Space Mono, uppercase, letter-spacing 0.2em, opacity 0.4. Always.
  • → Buttons must have at least one micro-interaction (shimmer, glow, conic, lift). No flat hover:bg only.
  • → When in doubt between light and dark: default to dark. Stone/Concrete (Theme C) for warmer projects.
  • → Section index numbers (01, 02, 03) use Syncopate at large size, low opacity. Always.
  • → No layout should feel symmetrical and safe. Use asymmetric grids, 1fr 1.2fr splits, col-span offsets.
